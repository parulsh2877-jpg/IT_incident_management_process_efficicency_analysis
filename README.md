# IT Incident Management Process Efficiency Analysis

## Overview
This project analyses incident management data to identify process inefficiencies and recommend improvements.

## Tools Used
- Excel
- SQLite
- Power BI
- Camunda

## Dataset
- 141,712 event records
- 24,918 incidents

## Key Findings
- Average Resolution Time: 7.36 days
- Reopen Rate: 1.10%
- Reassignment Rate: 43.63%
- Incidents with 20+ reassignments took ~40 days to resolve

## Deliverables
- Power BI Dashboard
- SQL Analysis
- BRD
- As-Is Process Map
- To-Be Process Map
